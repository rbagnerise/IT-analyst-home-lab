ARCHITECTURAL CONSTRAINTS — v16
══════════════════════════════════════════════════════
Host-to-DMZ Isolation (By Design)
   Dell host (10.10.30.2) cannot reach DMZ assets directly.
   OPT2 firewall rules permit only OPT2-internal → DMZ-VULN01 and
   block lateral movement to Servers/Clients. Host's role on VMnet19
   is IPv4 forwarding only, not DMZ interaction. Validate DMZ
   reachability from ATT-KALI01, never from the Dell host.

MacBook Kali Path Deprecated
   Cox Panoramic gateway enforces wireless client isolation with
   no UI toggle. Wireless clients (MacBook, Lenovo on Wi-Fi) cannot
   reach wired clients (Dell on ethernet). Phase 18 MacBook → Dell
   → DMZ path is blocked by ISP behavior the lab cannot control.
   ATT-KALI01 is now the canonical attack source for DMZ work.

VMware Repair Install Hazard
   Repair installs are destructive to virtual network configuration.
   They delete custom VMnets, may create phantom adapters on
   previously-occupied subnets (observed: phantom VMnet13 on
   10.10.30.0/24), and do not update VM-to-VMnet mappings even
   when the underlying VMnet is removed. Always export Virtual
   Network Editor config before running Repair, and verify all VM
   adapter slots after.
